# PHP Database Demo Project

Простой PHP проект, демонстрирующий подключение к базам данных MySQL и SQLite и создание таблицы.

## Описание

Этот проект создан в соответствии с требованиями [Issue #1](https://github.com/Dionisiy79/AI-sandbox/issues/1) и демонстрирует:

- ✅ Подключение к базе данных MySQL с именем `DataBase1`
- ✅ Подключение к базе данных SQLite в файле `DataBase1.sqlite`
- ✅ Создание пустой таблицы с именем `Table1`
- ✅ Универсальный контроллер для подключения, метаданных таблиц и CRUD-операций
- ✅ Правильную обработку ошибок
- ✅ Использование современного PDO интерфейса

## Требования

- PHP 7.4 или выше
- MySQL 5.7 или выше (или MariaDB 10.2+)
- Расширение PHP PDO с драйвером MySQL
- Расширение PHP PDO с драйвером SQLite

## Установка

### 1. Проверка требований

Убедитесь, что PHP и MySQL установлены:

```bash
php --version
mysql --version
```

Проверьте, что расширение PDO MySQL включено:

```bash
php -m | grep pdo_mysql
```

Проверьте, что расширение PDO SQLite включено:

```bash
php -m | grep pdo_sqlite
```

### 2. Настройка базы данных

Откройте файл `config.php` и настройте параметры подключения:

```php
return [
    'host' => 'localhost',        // Хост MySQL сервера
    'database' => 'DataBase1',    // Имя базы данных
    'username' => 'root',          // Имя пользователя MySQL
    'password' => '',              // Пароль пользователя MySQL
    // ...
];
```

### 3. Запуск скрипта

Выполните главный скрипт из командной строки:

```bash
php index.php
```

Или через веб-сервер, открыв `http://localhost/index.php` в браузере.

Для запуска SQLite-демонстрации выполните:

```bash
php index_sqlite.php
```

Скрипт создаст файл `DataBase1.sqlite` в корне проекта, если он еще не существует.

## Что делает скрипт

1. **Подключается к MySQL серверу** используя PDO
2. **Создает базу данных** `DataBase1` (если она не существует)
3. **Подключается к базе данных** `DataBase1`
4. **Создает таблицу** `Table1` со следующей структурой:
   - `id` - автоинкрементный первичный ключ
   - `created_at` - временная метка создания записи
5. **Проверяет** существование таблицы
6. **Отображает структуру** созданной таблицы

SQLite-демонстрация в `index_sqlite.php` выполняет аналогичные шаги для файла `DataBase1.sqlite`:

1. **Подключается к SQLite базе данных** используя PDO
2. **Создает файл базы данных** `DataBase1.sqlite` (если он не существует)
3. **Создает таблицу** `Table1` со следующей структурой:
   - `id` - автоинкрементный первичный ключ
   - `created_at` - временная метка создания записи
4. **Проверяет** существование таблицы
5. **Отображает структуру** созданной таблицы

## Универсальный контроллер

Файл `DatabaseController.php` содержит общий модуль для работы с разными типами баз данных. Сейчас он поддерживает:

- `mysql` / `pdo_mysql` - MySQL через PDO
- `sqlite` / `pdo_sqlite` - SQLite через PDO
- `mysqli` - MySQL через MySQLi

Контроллер предоставляет единый набор методов:

- `connect()` - подключение к выбранной базе данных
- `initializeDatabase()` - создание и выбор базы MySQL, если это требуется
- `createTable($table, $columns)` - создание таблицы
- `tableExists($table)` - проверка существования таблицы
- `describeTable($table)` - получение структуры таблицы
- `create($table, $data)` - добавление записи
- `read($table, $criteria)` - чтение записей
- `update($table, $data, $criteria)` - обновление записей
- `delete($table, $criteria)` - удаление записей
- `close()` - закрытие подключения

Пример CRUD-операций:

```php
require_once __DIR__ . '/DatabaseController.php';

$database = new DatabaseController('sqlite', [
    'file' => __DIR__ . '/DataBase1.sqlite',
]);

$database->connect();
$database->createTable('Users', [
    'id' => 'INTEGER PRIMARY KEY AUTOINCREMENT',
    'name' => 'TEXT NOT NULL',
]);

$id = $database->create('Users', ['name' => 'Alice']);
$rows = $database->read('Users', ['id' => $id]);
$database->update('Users', ['name' => 'Alice Smith'], ['id' => $id]);
$database->delete('Users', ['id' => $id]);
$database->close();
```

## Обработка ошибок

Скрипт использует блок `try-catch` для перехвата и обработки всех ошибок PDO. Если возникнет проблема с подключением или выполнением SQL, вы увидите сообщение об ошибке:

```
✗ Ошибка: SQLSTATE[HY000] [1045] Access denied for user 'root'@'localhost'
```

## Технические детали

### Использование PDO

Проект использует **PDO (PHP Data Objects)** - современный, безопасный интерфейс для работы с базами данных в PHP:

- Поддержка подготовленных запросов (защита от SQL-инъекций)
- Единый интерфейс для разных СУБД
- Объектно-ориентированный подход
- Надежная обработка ошибок через исключения

### SQLite

SQLite не требует отдельного сервера базы данных. Демонстрация использует файловую базу `DataBase1.sqlite`, которая создается автоматически при первом запуске `index_sqlite.php`.

### Безопасность

- Использование параметризованных запросов (где применимо)
- Включен режим исключений для ошибок БД
- Корректное закрытие соединений
- UTF-8 кодировка для поддержки международных символов

## Устранение неполадок

### Ошибка "could not find driver"

Убедитесь, что расширение PDO MySQL установлено:

```bash
# Ubuntu/Debian
sudo apt-get install php-mysql

# CentOS/RHEL
sudo yum install php-mysql

# После установки перезапустите веб-сервер
sudo systemctl restart apache2  # или nginx/php-fpm
```

Для SQLite установите расширение PHP SQLite:

```bash
# Ubuntu/Debian
sudo apt-get install php-sqlite3

# CentOS/RHEL
sudo yum install php-sqlite3
```

### Ошибка "Access denied"

Проверьте credentials в `config.php`. Убедитесь, что пользователь MySQL имеет права:

```sql
GRANT ALL PRIVILEGES ON DataBase1.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### Ошибка "Connection refused"

Убедитесь, что MySQL сервер запущен:

```bash
sudo systemctl status mysql  # или mariadb
sudo systemctl start mysql   # запустить, если не запущен
```
